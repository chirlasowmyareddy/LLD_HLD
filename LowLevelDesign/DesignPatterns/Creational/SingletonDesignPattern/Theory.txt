Singleton ensures that only one object of a class is created and provides a global access point to it.

How to implement (step by step)
    We need 3 things:
    1️⃣ Make constructor private
    2️⃣ Create a static instance
    3️⃣ Provide a public method to access it


Lazy + Thread Safe
class SingletonDBConnection {

    private static SingletonDBConnection instance;

    private SingletonDBConnection() {
        // private constructor
    }

    public static synchronized SingletonDBConnection getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }//this is very expensive

    //so we use double Locking
    public static SingletonDBConnection getInstance() {
        if (instance == null) {
            synchronized (SingletonDBConnection.class){
                if(instance == null){
                    instance = new SingletonDBConnection();
                }
            }
        }
        return instance;
    }

}



public class Main {
    public static void main(String[] args) {

        SingletonDBConnection s1 = SingletonDBConnection.getInstance();
        SingletonDBConnection s2 = SingletonDBConnection.getInstance();

        System.out.println(s1 == s2); // true
    }
}

Singleton pattern ensures only one instance of a class is created and provides a global access point. 
It is commonly used in scenarios like logging, configuration, and database connections.



4 ways to achive this:
    Eager
    Lazy
    Synchronized Method
    Double Locking(this is used in the industry)
